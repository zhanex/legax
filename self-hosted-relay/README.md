# Self-hosted Relay

English | [Simplified Chinese](README.zh-CN.md)

This folder contains the standalone deployable relay for Legax. Copy it to a Linux server and run `install.sh`; the installer installs Node.js when needed, creates a system user, writes a YAML config file with generated secrets, installs a service, and starts the relay. There is no environment-file layer — every value lives in the YAML.

Supported service managers:

- systemd: Debian, Ubuntu, Fedora, RHEL, CentOS Stream, Rocky Linux, AlmaLinux, openSUSE, Arch Linux
- OpenRC: Alpine Linux

Supported package managers for automatic Node.js installation:

- `apt-get`
- `dnf`
- `yum`
- `zypper`
- `pacman`
- `apk`

## Quick Install

For lightweight VPS, NAS, or Raspberry Pi installs where Node.js is already available:

```bash
npm install -g @legax/relay
legax-relay --config /path/to/config.yaml
```

The all-in-one `legax` package also includes `legax relay start`, but `@legax/relay` is smaller and is the recommended package for a relay-only server.

For managed Linux service installation:

```bash
cd self-hosted-relay
sudo ./install.sh
```

The installer prints:

- the YAML config file path that contains the generated secrets
- the relay web URL; browsers pair by scanning the QR code or entering the short code generated by `npm run daemon:pair` on the desktop
- service status commands

It does not print secrets by default. Browser access is paired with short one-time codes and QR payloads generated through the desktop pairing command.

## Custom Install

```bash
sudo SECRET="desktop-secret" \
  ./install.sh --host 0.0.0.0 --port 8787 --session default
```

(`SECRET` is read **only by the installer** when seeding the YAML; the running relay reads only the YAML.)

Common options:

```bash
sudo ./install.sh --install-dir /opt/legax-relay
sudo ./install.sh --config-dir /etc/legax-relay
sudo ./install.sh --data-dir /var/lib/legax-relay
sudo ./install.sh --no-start
sudo ./install.sh --no-node-install
```

## Installed Files

- `/opt/legax-relay/server.mjs`
- `/opt/legax-relay/lib/relay-server-core.mjs`
- `/opt/legax-relay/lib/yaml.mjs`
- `/opt/legax-relay/lib/paths.mjs`
- `/etc/legax-relay/config.yaml` (host / port / secret / store path / audit settings — all inline)
- `/var/lib/legax-relay/relay-store.json`
- `/etc/systemd/system/legax-relay.service` on systemd systems
- `/etc/init.d/legax-relay` on OpenRC systems

## Service Commands

systemd:

```bash
sudo systemctl status legax-relay
sudo systemctl restart legax-relay
sudo journalctl -u legax-relay -f
```

OpenRC:

```bash
sudo rc-service legax-relay status
sudo rc-service legax-relay restart
tail -f /var/log/legax-relay.log
```

## Security

- Keep `relay.secret` private; it authenticates the desktop or agent side.
- Pair browser devices from the desktop with `npm run daemon:pair`, then scan the printed QR code from the phone or enter the one-time code manually.
- Do not expose plain HTTP relay traffic on the public internet unless it is behind a trusted HTTPS reverse proxy or tunnel.
- Open TCP port `8787` only to trusted networks when possible.
- Rotate the secrets by editing `/etc/legax-relay/config.yaml` and restarting the service.

## Uninstall

```bash
sudo ./uninstall.sh
```

Remove config and data too:

```bash
sudo ./uninstall.sh --purge
```
