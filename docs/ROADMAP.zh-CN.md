# 路线图

[English](ROADMAP.md) | 简体中文

本路线图描述可能的优先级，不承诺具体发布日期。安全性、正确性和操作者控制优先于便利性。

## 近期

- 加强 relay 浏览器体验：更清晰的设备管理、配对反馈和离线诊断。
- 保持 Telegram 交互完整且低噪音，覆盖 CLI、项目或聊天、session 选择。
- 完善 Windows、macOS 和 Linux 的跨平台 daemon 设置文档。
- 扩展完整 Telegram 流程和浏览器 relay 流程的 E2E 覆盖。

## 中期

- 为更多暴露结构化 IO 的 Agent CLI 增加 adapter contract。
- 改进发布自动化和产物验证。
- 增加可选 CI 检查，覆盖打包和 Windows 执行。
- 补充 HTTPS relay 部署的反向代理示例。

## 非目标

- 自动批准原生 Agent 安全提示。
- 把 TUI 抓屏作为默认集成路径。
- 构建编程客户端、IDE、终端替代品、diff 工作台或完整 Web 编程环境。
- 把官方共享 relay 服务或官方共享 Telegram bot 作为默认部署模式。
